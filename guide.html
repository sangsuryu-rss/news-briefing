뉴스 브리핑 운영 가이드

세림전자 아침 뉴스 브리핑을 관리하는 방법입니다. 평소엔 자동으로 돌아가며, 공지 게시 등 필요할 때만 아래 순서를 따르세요.

1관리 주소 (즐겨찾기 권장)
📰
뉴스 사이트 (공유용)
sangsuryu-rss.github.io/news-briefing
📢
공지 올리기·내리기
…/edit/main/notice.txt
🔄
즉시 뉴스 갱신
…/actions/workflows/update.yml
📁
파일 올리기 (디자인·설정 변경)
…/upload/main
2공지사항 올리기 · 내리기 (가장 자주 쓰는 기능)
1
공지 올리기 주소 접속 → 연필(✏️) 편집 화면이 열립니다.
2
빈 칸에 공지 내용을 타이핑 (엔터로 여러 줄 가능).
3
오른쪽 위 Commit changes → 창에서 다시 Commit changes.
4
1~2분 뒤 사이트에서 Ctrl+Shift+R → 배너 오른쪽에 공지 표시 ✅
▸ 긴급 공지는 맨 앞에 긴급 또는 ! → 빨간색 강조.   ▸ 공지 내리기: 같은 화면에서 글자를 전부 지우고 Commit.
3지금 당장 최신 뉴스로 (수동 갱신)
1
즉시 갱신 주소 접속 → Run workflow → 초록 Run workflow 재클릭.
2
1~2분 뒤 사이트 강력새로고침(Ctrl+Shift+R) → 최신 뉴스 반영.
4꼭 기억할 규칙
🚫 로컬 파일 금지
내 컴퓨터의 index.html을 더블클릭하면 공지·날씨가 안 뜹니다. 항상 github.io 주소로 확인.
📤 업로드해야 반영
파일을 바꾸면 반드시 GitHub에 Commit(업로드) 해야 사이트에 적용됩니다.
⏱️ 반영엔 1~2분
바꾼 직후 안 보여도 실패 아님. 잠시 뒤 Ctrl+Shift+R.
5자동 운영 & 문제 해결
🕕 매일 아침 6~7시 자동 갱신
평소엔 손 안 대도 뉴스가 자동으로 최신화됩니다.
🛠️ 안 될 때
공지 안 보임 → 주소·Ctrl+Shift+R·2분 대기 / 그래도 이상하면 화면 캡처해 문의.
✏️ 주제·디자인 변경
"○○ 뉴스로 바꿔줘"라고 요청하면 파일을 만들어 드립니다.
